Git-summary

git

git이란

  • 버전 관리 시스템을 말하며 빠른 속도와 단순한 구조를 갖고 있다
    분산형 저장소를 지원하고, 비선형적 개발이 가능하다.

    git 시작전 작업

    config 작업 후, $ git config --list 명령으로 오류를 확인.

  • git의 프로세스의 전체적인 구조.

git 시작하기(방법 2가지)

1. git init

내 컴퓨터의 작업공간에서 github으로 소스코드를 내보내는 방법.

  1. 프로젝트 폴더 만들기.

  2. 폴더를 만둔 후, git init 명령으로 local repository 시작.

  3. github에 새로운 remote repository를 생성하고, 주소를 복사하여 local repository에 등록한다.

    • 프로젝트 폴더명과 생성한 repository 명을 갖게 한다.

  4. 새 파일을 만들고 작업을 수행한다.

  5. 작업 수행 후, $ git status 으로 git 상태 확인

  6. workspace(작업 공간)에 추가된 새로운 파일을 index로 staging 한다.

    • $ git add README.md

  7. 추가된 파일에 대한 설명을 commit

    • $ git commit(제목을 누구나 알 수 있게 작성해야 한다.)

  8. commit 후 remote repository로 push

    • 첫 commit이라면 git push -u origin master
    • 이후는, git push origin master

      2. git clone

      git에서 remote repository를 먼저 생성한 뒤, 작업공간으로 끌어와 작업을 수행.
    1. github에서 remote repository를 생성하되, LICENSE, README.md, .gitignore 등의 파일과 함께 생성,
      해당 repository의 주소를 복사한다.
    2. Teminal에서 $ git clone 복사한remoteRepo주소를 입력하여 작업공간에 local repository를 생성

    이하 과정은 init과 동일하다.
